Install

Include the PainSolver SDK and initialize once with your API base URL and board scope token. 

<script src="https://painsolver.vercel.app/sdk/painsolver.js"></script>
<div id="painsolver-board"></div>
<div id="painsolver-changelog"></div>
<script>
  PainSolver("init", {
    apiBaseUrl: "https://painsolver.vercel.app",
    boardToken: "SERVER_GENERATED_BOARD_TOKEN"
  });

  PainSolver("identify", {
    user: { email: "user@example.com", name: "User", appUserId: "u_1" },
    company: { name: "Acme", monthlySpend: 499 },
    hash: "SERVER_GENERATED_HMAC_SHA256"
  }).then(function () {
    PainSolver("render", { selector: "#painsolver-board" });
    PainSolver("initChangelog", { selector: "#painsolver-changelog" });
  });
</script>

Identify

Identify binds the visitor to a PainSolver user and company. This enables voting, changelog seen-state, and request attribution. 

PainSolver("identify", {
  user: {
    email: "pm@acme.com",
    name: "Product Manager",
    appUserId: "pm_42"
  },
  company: {
    name: "Acme",
    monthlySpend: 1200,
    healthStatus: "healthy",
    stripeCustomerId: "cus_123"
  },
  hash: "HMAC_SHA256_FROM_YOUR_BACKEND"
});

Company Data

Company data drives MRR-aware prioritization. Keep monthlySpend current to improve roadmap scoring.

  • Single company: pass company in identify.
  • Multiple company support payload shape is accepted and can be expanded in your backend integration.

Custom Fields

SDK accepts passthrough custom fields on user/company payloads. You can store these in your identity service, and map them to board access logic or analytics. 

user: {
  email: "ops@acme.com",
  customFields: {
    planTier: "enterprise",
    region: "us-east"
  }
},
company: {
  name: "Acme",
  customFields: {
    segment: "agency",
    arrBand: "50k-100k"
  }
}

Security

PainSolver supports both HMAC identify and signed short-lived tokens.

  • Identify hash: POST /api/v1/sdk/identify validates HMAC via PAINSOLVER_CLIENT_SECRET.
  • Board token: scope widget/changelog access to specific board(s).
  • SSO token: avoid exposing identity secrets in browser.

Advanced Identify

Use token-first flows for private boards and embedded environments.

// Backend (server-to-server): create SSO token
POST /api/v1/sdk/sso-token
Authorization: Bearer <API_KEY>
{
  "user": { "email": "user@acme.com", "name": "User" },
  "company": { "name": "Acme", "monthlySpend": 900 },
  "expiresInSeconds": 3600
}

// Frontend: authenticate with ssoToken
PainSolver("authenticate", { ssoToken: "..." });

Feedback Widget

Render the embeddable feedback list with board scoping and upvote actions.

PainSolver("render", {
  selector: "#painsolver-board",
  query: "comments sync"
});

Mobile Widget

For mobile apps, host a lightweight webview page that includes the same SDK snippet. Pass ssoToken and boardToken through query params generated by your backend.

Widget SSO

To avoid browser-side HMAC generation, use short-lived SSO tokens. 

PainSolver("init", {
  apiBaseUrl: "https://painsolver.vercel.app",
  boardToken: "...",
  ssoToken: "..."
});

PainSolver("authenticate", { ssoToken: "..." })
  .then(function () {
    return PainSolver("render", { selector: "#painsolver-board" });
  });

Widget Advanced

  • onLoadCallback in init for instrumentation.
  • boardToken to enforce private board scope.
  • hasUnseenEntries to drive badges and notification dots.
PainSolver("init", {
  apiBaseUrl: "https://painsolver.vercel.app",
  boardToken: "...",
  onLoadCallback: function (event) {
    console.log("PainSolver loaded", event.type, event.count);
  }
});

Changelog Widget

Render release updates and track per-user seen status.

PainSolver("initChangelog", {
  selector: "#painsolver-changelog",
  limit: 10
});

PainSolver("hasUnseenEntries", {
  callback: function (hasUnseen, meta) {
    console.log("Unseen?", hasUnseen, meta.unseenCount);
  }
});

Changelog Customization

Use CSS overrides scoped to .ps-root to match your app design. You can also render your own shell and call SDK APIs directly.

Changelog Privacy

Use board tokens for private release channels. Without a valid token, private board changelog entries are not returned.
