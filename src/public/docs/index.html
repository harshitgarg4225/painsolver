Install

Include the PainSolver SDK and initialize once with your API base URL and board scope token. 

<script src="https://painsolver.vercel.app/sdk/painsolver.js"></script>
<div id="painsolver-board"></div>
<div id="painsolver-changelog"></div>
<script>
  PainSolver("init", {
    apiBaseUrl: "https://painsolver.vercel.app",
    boardToken: "SERVER_GENERATED_BOARD_TOKEN"
  });

  PainSolver("identify", {
    user: { email: "user@example.com", name: "User", appUserId: "u_1" },
    company: { name: "Acme", monthlySpend: 499 },
    hash: "SERVER_GENERATED_HMAC_SHA256"
  }).then(function () {
    PainSolver("render", { selector: "#painsolver-board" });
    PainSolver("initChangelog", { selector: "#painsolver-changelog" });
  });
</script>

React / Next.js Installation

For React and Next.js apps, create a reusable component and load the SDK dynamically.

1. Create the FeedbackWidget component

// components/FeedbackWidget.tsx
'use client'; // For Next.js App Router

import { useEffect, useRef } from 'react';

declare global {
  interface Window {
    PainSolver: (action: string, options?: any) => Promise<any>;
  }
}

interface FeedbackWidgetProps {
  boardToken: string;
  user?: {
    email: string;
    name: string;
    appUserId?: string;
  };
  company?: {
    name: string;
    monthlySpend?: number;
  };
  hash?: string;
}

export function FeedbackWidget({ boardToken, user, company, hash }: FeedbackWidgetProps) {
  const containerRef = useRef<HTMLDivElement>(null);
  const initialized = useRef(false);

  useEffect(() => {
    if (initialized.current) return;
    initialized.current = true;

    // Load SDK script
    const script = document.createElement('script');
    script.src = 'https://painsolver.vercel.app/sdk/painsolver.js';
    script.async = true;
    script.onload = async () => {
      // Initialize
      await window.PainSolver('init', {
        apiBaseUrl: 'https://painsolver.vercel.app',
        boardToken
      });

      // Identify user if provided
      if (user && hash) {
        await window.PainSolver('identify', { user, company, hash });
      }

      // Render widget
      if (containerRef.current) {
        await window.PainSolver('render', { 
          selector: `#${containerRef.current.id}` 
        });
      }
    };
    document.body.appendChild(script);

    return () => {
      document.body.removeChild(script);
    };
  }, [boardToken, user, company, hash]);

  return <div id="painsolver-feedback" ref={containerRef} />;
}

2. Use in your app

// app/feedback/page.tsx (Next.js) or App.tsx (React)
import { FeedbackWidget } from '@/components/FeedbackWidget';

export default function FeedbackPage() {
  // Get these from your auth context
  const user = { email: 'user@example.com', name: 'John Doe' };
  const company = { name: 'Acme Inc', monthlySpend: 499 };
  const hash = 'YOUR_HMAC_FROM_API'; // Generate server-side!

  return (
    <div className="container mx-auto py-8">
      <h1>Feature Requests</h1>
      <FeedbackWidget
        boardToken="YOUR_BOARD_TOKEN"
        user={user}
        company={company}
        hash={hash}
      />
    </div>
  );
}

3. Generate HMAC server-side (API route)

// app/api/painsolver-hash/route.ts (Next.js App Router)
import { createHmac } from 'crypto';
import { NextResponse } from 'next/server';

export async function POST(request: Request) {
  const { email } = await request.json();
  
  const hash = createHmac('sha256', process.env.PAINSOLVER_CLIENT_SECRET!)
    .update(email)
    .digest('hex');

  return NextResponse.json({ hash });
}

Vue.js Installation

For Vue apps, create a composable and component to manage the SDK lifecycle.

1. Create the composable

// composables/usePainSolver.ts
import { ref, onMounted, onUnmounted } from 'vue';

declare global {
  interface Window {
    PainSolver: (action: string, options?: any) => Promise<any>;
  }
}

export function usePainSolver(boardToken: string) {
  const isLoaded = ref(false);
  let script: HTMLScriptElement | null = null;

  onMounted(() => {
    script = document.createElement('script');
    script.src = 'https://painsolver.vercel.app/sdk/painsolver.js';
    script.async = true;
    script.onload = async () => {
      await window.PainSolver('init', {
        apiBaseUrl: 'https://painsolver.vercel.app',
        boardToken
      });
      isLoaded.value = true;
    };
    document.body.appendChild(script);
  });

  onUnmounted(() => {
    if (script) document.body.removeChild(script);
  });

  const identify = async (user: any, company: any, hash: string) => {
    if (!isLoaded.value) return;
    await window.PainSolver('identify', { user, company, hash });
  };

  const render = async (selector: string) => {
    if (!isLoaded.value) return;
    await window.PainSolver('render', { selector });
  };

  return { isLoaded, identify, render };
}

2. Create the component

<!-- components/FeedbackWidget.vue -->
<script setup lang="ts">
import { watch } from 'vue';
import { usePainSolver } from '@/composables/usePainSolver';

const props = defineProps<{
  boardToken: string;
  user?: { email: string; name: string };
  company?: { name: string; monthlySpend?: number };
  hash?: string;
}>();

const { isLoaded, identify, render } = usePainSolver(props.boardToken);

watch(isLoaded, async (loaded) => {
  if (!loaded) return;
  
  if (props.user && props.hash) {
    await identify(props.user, props.company, props.hash);
  }
  await render('#painsolver-widget');
});
</script>

<template>
  <div id="painsolver-widget">
    <p v-if="!isLoaded">Loading feedback widget...</p>
  </div>
</template>

Node.js Backend Setup

Generate secure HMAC hashes and board tokens server-side to protect your client secret.

1. Install dependencies

npm install crypto

2. HMAC generation utility

// utils/painsolver.ts
import crypto from 'crypto';

const CLIENT_SECRET = process.env.PAINSOLVER_CLIENT_SECRET!;
const API_KEY = process.env.PAINSOLVER_API_KEY!;

/**
 * Generate HMAC hash for user identification
 * Pass this to the frontend for the identify() call
 */
export function generateIdentifyHash(email: string): string {
  return crypto
    .createHmac('sha256', CLIENT_SECRET)
    .update(email)
    .digest('hex');
}

/**
 * Generate a board-scoped token for the widget
 */
export async function generateBoardToken(boardId: string): Promise<string> {
  const response = await fetch('https://painsolver.vercel.app/api/v1/sdk/board-token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({ boardId })
  });
  
  const data = await response.json();
  return data.token;
}

/**
 * Create an SSO token for secure authentication
 */
export async function createSSOToken(user: {
  email: string;
  name: string;
}, company?: {
  name: string;
  monthlySpend?: number;
}): Promise<string> {
  const response = await fetch('https://painsolver.vercel.app/api/v1/sdk/sso-token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({
      user,
      company,
      expiresInSeconds: 3600
    })
  });
  
  const data = await response.json();
  return data.token;
}

3. Express.js API endpoint example

// routes/painsolver.ts
import { Router } from 'express';
import { generateIdentifyHash, generateBoardToken } from '../utils/painsolver';

const router = Router();

// Endpoint for frontend to get HMAC hash
router.post('/hash', async (req, res) => {
  const { email } = req.body;
  
  if (!email) {
    return res.status(400).json({ error: 'Email required' });
  }
  
  const hash = generateIdentifyHash(email);
  res.json({ hash });
});

// Endpoint to get board token
router.post('/board-token', async (req, res) => {
  const { boardId } = req.body;
  
  try {
    const token = await generateBoardToken(boardId);
    res.json({ token });
  } catch (error) {
    res.status(500).json({ error: 'Failed to generate token' });
  }
});

export default router;

Company Data

Company data drives MRR-aware prioritization. Keep monthlySpend current to improve roadmap scoring.

  • Single company: pass company in identify.
  • Multiple company support payload shape is accepted and can be expanded in your backend integration.

  }
}

Security

PainSolver supports both HMAC identify and signed short-lived tokens.

  • Identify hash: POST /api/v1/sdk/identify validates HMAC via PAINSOLVER_CLIENT_SECRET.
  • Board token: scope widget/changelog access to specific board(s).
  • SSO token: avoid exposing identity secrets in browser.

Advanced Identify

Use token-first flows for private boards and embedded environments.

Stripe Integration

PainSolver can automatically fetch customer MRR from Stripe. When a user is identified, their email is used to look up active subscriptions and calculate normalized monthly revenue.

Configuration

Set the following environment variables:

# Enable Stripe integration
STRIPE_API_KEY=sk_live_your_stripe_secret_key

# Set to "false" to use real Stripe (defaults to mock mode)
USE_MOCK_STRIPE=false

How it works

  • When identify() is called, PainSolver looks up the customer by email in Stripe
  • Active subscriptions are fetched and normalized to monthly values
  • Yearly plans are divided by 12, quarterly by 3, etc.
  • The total MRR is stored on the company record for prioritization

MRR & Revenue Data

MRR data powers PainSolver's revenue-weighted prioritization. You can provide MRR in multiple ways:

Option 1: Pass manually via SDK

PainSolver("identify", {
  user: { email: "user@acme.com", name: "User" },
  company: {
    name: "Acme Corp",
    monthlySpend: 1200,  // Direct MRR value in dollars
    stripeCustomerId: "cus_123"  // Optional: for reference
  },
  hash: "HMAC_SHA256_FROM_YOUR_BACKEND"
});

Option 2: Automatic Stripe lookup

If Stripe is configured, MRR is automatically calculated from active subscriptions when monthlySpend is not provided.

Option 3: Server-side API

// Update company MRR via API
PUT /api/v1/companies/:companyId
Authorization: Bearer <API_KEY>
{
  "monthlySpend": 2500
}

MRR in Prioritization

Ideas are scored using total MRR from all voters. High-MRR customers' requests bubble up in the "Trending" sort and appear prominently in the roadmap planning view.
